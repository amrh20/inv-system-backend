📦 Item Master — System Documentation

OS&E Cloud Inventory System — Module Reference

1. Purpose

The Item Master is the central registry for all OS&E (Operating Supplies & Equipment) items. Every item that will be received, stored, issued, transferred, counted, or reported on must first exist in the Item Master.

Single Source of Truth for: item identity, classification, pricing, supplier linkage, and unit of measure configuration.

2. Key Features

Feature Description
➕ Create / Edit Items Full form with name, barcode, category, supplier, department, pricing
📷 Image Management Per-item image upload + bulk ZIP upload matched by barcode
📐 Unit of Measure BASE / PURCHASE / ISSUE units with conversion rates
📥 Excel Import Upload items via Excel with preview & validation wizard
📤 Excel Export Export filtered item list to .xlsx
🔍 Search & Filter By name, barcode, category, active/inactive status
🔄 Activate / Deactivate Soft delete — item stays in history but hidden from operations

3. Item Fields

Reorder Qty Optional Suggested order quantity when reorder is triggered
Description Optional Free-text notes

4. Unit Structure Lock 🔒

Unit modifications (BASE unit, conversion rates, unit structure) are locked when stock exists in any location.

❌ Stock > 0 → BLOCKED
"Cannot modify item units while stock exists. Please zero out stock via adjustment/count or create a new item."
✅ Stock = 0 → ALLOWED
Unit changes are permitted when no stock exists in any location.

Why this policy?

  • Changing units with active stock corrupts inventory valuation
  • Historical quantities become misinterpreted (e.g., 50 "Pieces" becomes 50 "Cartons")
  • Consumption and variance reports produce incorrect results
  • This follows the same standard used by SAP, Oracle, and Odoo

6. Built-in Controls

Control Type Description
Duplicate Name Check Validation No two items can share the same name
Duplicate Barcode Check Validation No two items can share the same barcode
Department → Store Filter UI Logic Store dropdown is filtered by selected department
Category → Subcategory UI Logic Dependent dropdown relationship
Unit Structure Lock Business Rule Cannot modify units while stock > 0
Delete Protection Business Rule Cannot delete item with active stock
Audit Trail Compliance All changes logged with before/after values

7. Audit & Compliance

  • ✅ Immutable Audit Log — every create, update, and delete is recorded
  • ✅ Before / After Snapshots — full JSON diff of what changed
  • ✅ User Attribution — who made the change, when, from where
  • ✅ Non-editable — audit records cannot be modified or deleted
  • ✅ Accessible via Sidebar → Admin → Audit Log
